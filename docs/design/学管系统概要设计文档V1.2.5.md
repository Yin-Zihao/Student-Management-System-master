## 1 引言

### 1.1 项目简要介绍

本项目旨在开发一个面向计算机学院的学生成绩管理系统，实现学生信息管理、课程管理、成绩录入与查询、学分统计、成绩排名及简单数据可视化等功能。系统面向管理员、教师、学生、人事秘书四个角色，为学院教学、科研、人事、党务管理提供数字化支持。

### 1.2 项目背景

当前计算机学院的学生成绩管理主要依赖 Excel 表格或分散的教务系统，存在数据不集中、查询不便、统计分析困难、科研人事数据无法统一管理等问题。为提升教学管理效率，方便学生及时了解学业进度，满足学院科研、人事、竞赛、文件管理需求，本系统应运而生。

### 1.3 项目的创新点

- **成绩可视化**：用图表展示班级成绩分布、个人成绩趋势
- **Excel 批量导入导出**：支持学生、课程、成绩、科研、竞赛、人事数据导入导出
- **学分进度预警**：自动计算学分并对比毕业要求
- **教职工信息完整度校验**：支撑幼儿费用、上学等福利服务
- **党务文件全生命周期管理**：支持废改立状态管理

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 学分预警 | 当学生已获学分低于毕业要求时，系统自动标记为预警状态 |
| 数据看板 | 使用图表展示关键统计指标的仪表盘页面 |
| 数据来源 | 学院统计 / 学校下发 |


## 2 任务概述

### 2.1 目标

开发一个功能完整、界面友好的学生成绩管理系统，实现以下目标：

1. **用户管理**：支持管理员、教师、学生、人事秘书四类角色的登录与权限控制；管理员可维护用户信息
2. **基础数据管理**：维护学院、专业、班级的层级结构；维护课程基本信息（课程名、学分、授课教师）
3. **成绩管理**：支持教师录入、修改、查询所授课程的成绩；支持学生查询个人成绩；支持管理员查看全部成绩
4. **统计与分析**：自动计算学生总学分；按专业/年级计算成绩排名；通过图表展示各班级成绩分布、不及格率等
5. **数据导入导出**：支持通过 Excel 文件导入学生、课程、成绩、竞赛、科研、人事数据；支持查询结果导出为 Excel
6. **科研管理**：实现论文、科研项目、科研奖励、教研奖励、大创项目的统一管理
7. **人事管理**：实现教职工信息、学历学位、年度考核、创业信息、子女信息、特聘与新入职教师信息管理
8. **党务管理**：实现学院党字、院字、学字文件的录入、查询、状态维护与关联管理

## 3 总体设计

### 3.1 基本设计概念和处理流程

系统采用 **B/S架构**，前后端分离：

- 后端基于 **Spring Boot** 提供 RESTful API
- 前端使用 **Vue + Element Plus** 构建用户界面
- 数据存储采用 **MySQL** 数据库

**核心处理流程**：

1. 用户通过浏览器访问系统，输入用户名密码登录
2. 后端验证身份，返回JWT令牌，前端保存令牌用于后续请求
3. 根据用户角色，前端展示不同的功能菜单
4. 用户进行数据操作（增删改查），前端调用对应API，后端处理请求并返回结果
5. 对于统计看板，后端定时或实时聚合数据，前端通过图表展示
6. 对于文件上传、信息维护等操作，后端统一校验并记录操作日志

### 3.2 系统架构

系统整体分为三层：

| 层级 | 技术栈 | 职责 |
|------|--------|------|
| **前端层** | Vue3单页应用 + Axios | 界面渲染和用户交互，调用后端API |
| **后端层** | Spring Boot（Controller/Service/Mapper） | 业务逻辑处理、数据库交互 |
| **数据层** | MySQL数据库 + Redis（可选） | 数据存储、会话管理 |

## 6 安全与权限设计

1. **身份认证**：采用 JWT 令牌机制，用户登录后获取令牌，后续请求携带令牌进行身份验证
2. **密码加密**：用户密码采用 BCrypt 加密存储，确保密码安全
3. **角色权限**：按管理员、教师、学生、人事秘书分配不同操作权限，不同角色只能访问授权功能
4. **数据隔离**：学生仅可查看个人成绩，教师仅可管理所授课程与个人信息，人事秘书可管理全院人事与文件
5. **操作日志**：对重要数据的增删改操作自动记录日志，便于追溯
6. **敏感信息保护**：教职工人事信息、子女信息、证书文件仅本人、管理员与人事秘书可访问
7. **文件权限**：上传的证书、合同等文件设置访问权限，防止未授权下载
