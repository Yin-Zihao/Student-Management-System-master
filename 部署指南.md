# 学管系统 Linux 部署指南

## 部署方式选择

本项目提供两种部署方式：
1. **Docker Compose 部署**（推荐）- 简单快速，适合生产环境
2. **手动部署** - 适合开发环境或需要自定义配置的场景

---

## 方式一：Docker Compose 部署（推荐）

### 前置要求

- Linux 服务器（CentOS 7+ / Ubuntu 18.04+）
- Docker 20.10+
- Docker Compose 2.0+
- 至少 2GB 可用内存
- 开放端口：8091（访问端口）、3306（数据库，可选）

### 步骤 1: 上传代码到服务器

#### 方法 A: 使用 SCP 命令（从 Windows/Mac/Linux）

```bash
# 在本地电脑执行
scp -r Student-Management-System-master root@your-server-ip:/opt/
```

#### 方法 B: 使用 Git 克隆

```bash
# 在服务器上执行
cd /opt
git clone <你的 Git 仓库地址> Student-Management-System-master
```

#### 方法 C: 使用 FTP/SFTP 工具

使用 FileZilla、WinSCP 等工具将整个项目文件夹上传到 `/opt/` 目录

### 步骤 2: 安装 Docker 和 Docker Compose

#### Ubuntu/Debian 系统

```bash
# 更新包索引
sudo apt update

# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

#### CentOS/RHEL 系统

```bash
# 安装 Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install docker-ce docker-ce-cli containerd.io -y
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 步骤 3: 配置项目

```bash
# 进入项目目录
cd /opt/Student-Management-System-master

# 创建 nginx 配置目录
mkdir -p nginx

# 复制 nginx 配置文件
cp nginx.conf nginx/default.conf

# 编辑环境变量文件（修改数据库密码等）
vi .env
```

修改 `.env` 文件中的配置：
```bash
DB_PASSWORD=你的强密码
MYSQL_ROOT_PASSWORD=你的强密码
```

### 步骤 4: 构建并启动服务

```bash
# 赋予部署脚本执行权限
chmod +x deploy.sh

# 执行部署脚本
./deploy.sh
```

或者手动执行：

```bash
# 构建前端
cd code/frontend
npm install
npm run build
cd ../..

# 启动所有服务
docker-compose up -d

# 查看启动日志
docker-compose logs -f
```

### 步骤 5: 验证服务

```bash
# 查看所有容器状态
docker-compose ps

# 应该看到三个运行中的容器：
# - sms-nginx (8091 端口)
# - sms-backend (8080 端口)
# - sms-mysql (3306 端口)
```

### 步骤 6: 访问系统

打开浏览器访问：
```
http://你的服务器IP:8091
```

## 方式二：手动部署（适合开发环境）

### 前置要求

- Java 17+
- Node.js 18+
- MySQL 8.0+
- Nginx

### 步骤 1: 安装依赖

#### 安装 Java 17

```bash
# Ubuntu/Debian
sudo apt install openjdk-17-jdk -y

# CentOS/RHEL
sudo yum install java-17-openjdk-devel -y

# 验证
java -version
```

#### 安装 Node.js 18

```bash
# 使用 NodeSource
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs

# 或使用 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18
nvm use 18

# 验证
node -v
npm -v
```

#### 安装 MySQL 8.0

```bash
# Ubuntu/Debian
sudo apt install mysql-server -y

# CentOS/RHEL
sudo yum install mysql-server -y

# 启动 MySQL
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 安全配置
sudo mysql_secure_installation
```

#### 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt install nginx -y

# CentOS/RHEL
sudo yum install nginx -y

# 启动 Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

### 步骤 2: 配置数据库

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库和用户
CREATE DATABASE student_management CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'sms_user'@'localhost' IDENTIFIED BY '你的密码';
GRANT ALL PRIVILEGES ON student_management.* TO 'sms_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 步骤 3: 构建前端

```bash
cd /opt/Student-Management-System-master/code/frontend

# 安装依赖
npm install

# 构建生产版本
npm run build

# 构建完成后，dist 目录包含静态文件
```

### 步骤 4: 配置后端

编辑 `code/backend/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/student_management?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: sms_user
    password: 你的密码
```

### 步骤 5: 打包后端

```bash
cd /opt/Student-Management-System-master/code/backend

# 使用 Maven 打包
mvn clean package -DskipTests

# 生成的 jar 文件在 target/ 目录
# target/sms-backend.jar
```

### 步骤 6: 配置 Nginx

编辑 Nginx 配置文件：

```bash
sudo vi /etc/nginx/conf.d/sms.conf
```

添加以下配置：

```nginx
server {
    listen 8091;
    server_name localhost;

    # 前端静态文件
    location / {
        root /opt/Student-Management-System-master/code/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

测试配置并重载 Nginx：

```bash
# 测试配置
sudo nginx -t

# 重载配置
sudo systemctl reload nginx
```

### 步骤 7: 启动后端服务

```bash
cd /opt/Student-Management-System-master/code/backend

# 后台运行
nohup java -jar target/sms-backend.jar --server.port=8080 > backend.log 2>&1 &

# 或使用 systemd 创建服务
sudo vi /etc/systemd/system/sms-backend.service
```

添加以下内容：

```ini
[Unit]
Description=Student Management System Backend
After=syslog.target network.target

[Service]
User=root
WorkingDirectory=/opt/Student-Management-System-master/code/backend
ExecStart=/usr/bin/java -jar target/sms-backend.jar --server.port=8080
SuccessExitStatus=143
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start sms-backend
sudo systemctl enable sms-backend
sudo systemctl status sms-backend
```

### 步骤 8: 验证服务

```bash
# 检查后端是否运行
curl http://localhost:8080/api/health

# 检查 Nginx 是否运行
curl http://localhost:8091

# 查看防火墙状态
sudo firewall-cmd --list-all

# 如果防火墙开启，添加 8091 端口
sudo firewall-cmd --permanent --add-port=8091/tcp
sudo firewall-cmd --reload
```

## 常见问题排查

### 1. 容器无法启动

```bash
# 查看详细日志
docker-compose logs nginx
docker-compose logs backend
docker-compose logs mysql

# 检查端口占用
netstat -tlnp | grep 8091
netstat -tlnp | grep 8080
netstat -tlnp | grep 3306
```

### 2. 前端无法访问后端

检查 CORS 配置和 Nginx 代理配置：
- 确保 Nginx 的 `/api/` location 正确代理到后端
- 检查后端 CORS 配置允许跨域

### 3. 数据库连接失败

```bash
# 检查 MySQL 是否运行
docker-compose ps mysql

# 查看数据库日志
docker-compose logs mysql

# 测试数据库连接
docker-compose exec backend ping -c 4 mysql
```

### 4. 防火墙问题

```bash
# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=8091/tcp
sudo firewall-cmd --reload

# Ubuntu/Debian
sudo ufw allow 8091/tcp
sudo ufw enable
```

### 5. 权限问题

```bash
# 修复文件权限
sudo chown -R root:root /opt/Student-Management-System-master
sudo chmod -R 755 /opt/Student-Management-System-master
```

## 常用运维命令

### Docker 方式

```bash
# 查看服务状态
docker-compose ps

# 查看实时日志
docker-compose logs -f

# 重启所有服务
docker-compose restart

# 重启单个服务
docker-compose restart nginx

# 停止所有服务
docker-compose down

# 停止并删除所有数据（谨慎使用）
docker-compose down -v

# 重新构建并启动
docker-compose up -d --build
```

### 手动部署方式

```bash
# 查看后端服务状态
sudo systemctl status sms-backend

# 重启后端
sudo systemctl restart sms-backend

# 查看后端日志
journalctl -u sms-backend -f

# 重载 Nginx
sudo systemctl reload nginx

# 重启 Nginx
sudo systemctl restart nginx

# 重启 MySQL
sudo systemctl restart mysqld
```

## 安全建议

1. **修改默认密码**
   - 数据库 root 密码
   - 应用数据库用户密码
   - 使用强密码（大小写字母 + 数字 + 特殊字符，至少 12 位）

2. **配置防火墙**
   ```bash
   # 只开放必要端口
   sudo firewall-cmd --permanent --add-port=8091/tcp
   sudo firewall-cmd --permanent --remove-service=mysql
   sudo firewall-cmd --reload
   ```

3. **启用 HTTPS（生产环境必需）**
   ```bash
   # 使用 Let's Encrypt 免费证书
   sudo apt install certbot python3-certbot-nginx -y
   sudo certbot --nginx -d yourdomain.com
   ```

4. **定期备份数据库**
   ```bash
   # 创建备份脚本
   mysqldump -u root -p student_management > backup_$(date +%Y%m%d_%H%M%S).sql
   ```

5. **监控系统资源**
   ```bash
   # 安装监控工具
   sudo apt install htop iotop -y
   
   # 查看 Docker 资源使用
   docker stats
   ```

## 升级指南

### Docker 方式升级

```bash

cd /opt/Student-Management-System-master
git pull

# 重新构建并启动

docker-compose up -d --build
```

### 手动方式升级

```bash
# 更新前端
cd /opt/Student-Management-System-master/code/frontend
git pull
npm install
npm run build

# 更新后端
cd /opt/Student-Management-System-master/code/backend
git pull
mvn clean package -DskipTests
sudo systemctl restart sms-backend
```
